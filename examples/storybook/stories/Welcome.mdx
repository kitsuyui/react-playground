This Storybook is organized by Web-compatible roles rather than by package name alone.

## Taxonomy

- `Semantic Wrappers`
- `State Providers`
- `Interaction Primitives`
- `Visualization Primitives`
- `Layout Primitives`
- `Default UIs`

## Reading The Catalog

Each package should expose its lowest-level reusable primitive first.
If a package also includes an auto wrapper or a default UI, those stories are listed separately so the layering stays visible.

## Intended Use

- Primitive stories show reusable contracts.
- Provider stories show state ownership and composition boundaries.
- Default UI stories show replaceable reference implementations.
- Overview pages explain how those layers relate to each other.
